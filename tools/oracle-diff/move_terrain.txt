# 移動範囲ハーネス検証コーパス (TERRAIN)。地形コスト + 移動タイプ通行可否を探る。
# 11x7 マップ。左半分に地上ユニット (陸)、右半分に飛行ユニット (空陸) を「別セル」に置き、
# 各々の周囲に地形を仕込む。1 マップ 2 ユニットだが start セルを離して置くため StandBy の
# ずれ (占有回避の再配置) を避けられる。
#
# 採用地形 id はこのシナリオの terrain.txt 実値で検証済み:
#   0  = 平地 plain   陸  move_cost 1
#   11 = 林 woods      陸  move_cost 1.5  (Rust ceil 2 → 2x=4 / C# 半端保持 → 2x=3)  EXPECT cost 乖離
#   15 = 山 mountain   陸  move_cost 2.5  (Rust ceil 3 → 2x=6 / C# → 2x=5)            EXPECT cost 乖離
#   17 = 海 sea        水  move_cost 2    (進入可否は水適応に依存)
#   注: プロンプトの「4=海」は誤り。このファイルでは id 4 = 空(sky)。海は id 17。
#   注: マジンガーＺ adaption=-ACB → 水適応 C を持つため海は BLOCKED ではなく
#       進入可 (cost 2 game = 4 in 2x)。両エンジン一致 (= 水適応 passability の検証)。
#       「水適応なしの地上ユニットが海で BLOCKED」を厳密に検証するには水適応 '-' の
#       ユニットが要る (このコーパスでは未カバー = 既知ギャップ)。
@map 11 7
# --- 左: 地上ユニット マジンガーＺ (陸; 水適応C) を (2,3) に。周囲に林/山/海 ---
@cell 2 2 11   # 林 woods (地上ユニットの真上)
@cell 2 4 15   # 山 mountain (地上ユニットの真下)
@cell 1 3 17   # 海 sea (地上ユニットの左) — 水適応Cのため進入可 (両エンジン一致)
@unit マジンガーＺ 0 味方 人工知能 10 2 3
@move マジンガーＺ
# --- 右: 飛行ユニット マジンカイザー(強化型) (空陸) を (8,3) に。隣に海 ---
@cell 8 2 17   # 海 sea (飛行ユニットの真上) — 空ユニットは飛び越えられるか?
@cell 8 4 11   # 林 woods (飛行ユニットの真下) — 空移動なら林コストは平地相当?
@unit マジンカイザー(強化型) 0 味方 人工知能 10 8 3
@move マジンカイザー(強化型)
